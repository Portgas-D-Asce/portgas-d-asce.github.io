树状数组原本是均衡版的前缀和数组，改造后在 rmq 问题中扮演着重要角色，属于是轻武器： 虽然使用受限，但效率高，常用于解决简单的的 “单点更新 + 区间查询” 问题。

注意：本文在对于原数组 nums，理解原理时，是 1-indexed 的，代码实现时，是 0-indexed 的（毕竟作为 api，要好用）

原数组 & 前缀和数组 & 差分数组 & 树状数组

设有原数组（常数时间内访问任意元素） nums:

  • 数组 sum[i] = nums[0] + nums[1] + ... + nums[i] 称为 nums 的前缀和数组（常数时间获取任意区间的元素和）
  • 数组 dif[i] = nums[i] - nums[i - 1] 称为 nums 的差分数组（常数时间内对任意区间内元素增加/减少固定数值）

前缀和、差分互为逆运算：

  • 原数组 是 差分数组 的前缀和数组！！！
  • 原数组 是 前缀和数组 的差分数组！！！

前缀和数组、差分数组、树状数组本质上都是原数组的某种变换或表示，通过适当的操作可以相互转换

前缀和数组： 直接存储了前缀和信息，查询效率高，更新效率低，是一种 “极端” 的数据结构

  • 构建时间复杂度：$O(n)$；
  • 查询原数组任意区间内元素的和： 两个前缀和作差，时间复杂度 $O(1)$；
  • 修改原数组中某个元素的值需要对前缀和数组进行维护： 时间复杂度为 $O(n)$；

树状数组： 间接存储了前缀和信息，被分为连续的多段，查询效率略低，更新效率较高，是一种

  • 构建时间复杂度： $O(n)$；
  • 查询原数组任意区间内元素的和： 两个前缀和作差，时间复杂度 $O(logn)$；
  • 修改原数组中某个元素的值需要对树状数组进行维护： 时间复杂度为 $O(logn)$；

如上图所示，树状数组 ft 的长度与原数组 nums （1-indexed）的长度关系：ft.length() = nums.length() + 1

  • ft[0] = 0，用于辅助计算，不存储原数组的信息
  • ft[1] = nums[1]
  • ft[2] = nums[2] + nums[1]
  • ft[3] = nums[3]
  • ft[4] = nums[4] + ft[3] + ft[2] = nums[4] + nums[3] + nums[2] + nums[1]
  • ft[5] = nums[5]
  • ft[6] = nums[6] + nums[5]
  • ft[7] = nums[7]
  • ft[8] = nums[8] + ft[7] + ft[6] + ft[4] = nums[8] + nums[7] + ... + nums[1]
  • ……

根据树状数组 ft 显然也能还原出原数组 nums:

  • nums[1] = ft[1]
  • nums[2] = ft[2] - ft[1]
  • nums[3] = ft[3]
  • nums[4] = ft[4] - ft[3] - ft[2]
  • nums[5] = ft[5]
  • nums[6] = ft[6] - ft[5]
  • nums[7] = ft[7]
  • nums[8] = ft[8] - ft[7] - ft[6] - ft[4]
  • ……

树状数组（普通版）

template<typename T>
class FenwickTree {
public:
    FenwickTree(int n) : _ft(std::vector<T>(n + 1, 0)) {}
    
    // 0-indexed nums[idx] += x
    void add(int idx, const T& x) {
        idx++;
        while(idx < _ft.size()) {
            _ft[idx] += x;
            idx += lowbit(idx);
        }
    }
    
    // 0-indexed prefix sum [0, idx]
    T query(int idx) const {
        idx++;
        T res = 0;
        while(idx) {
            res += _ft[idx];
            idx -= lowbit(idx);
        }
        return res;
    }
    
    // 0-indexed range sum [p, r]
    T query(int p, int r) const {
        return prefix(++r) - prefix(p);
    }
private:
    static int lowbit(int x) { return x & (-x); }
private:
    vector<T> _ft;
};

lowbit 函数是什么？

lowbit(x) 表示 x 的二进制表示最后一位 1 的位权

x 的二进制表示为：0...1000000
x 的反码为：     1...0111111
x 的补码为：     1...1000000（也就是 -x 的二进制表示）
x & (-x)：      0...1000000

ft[i] 的含义是什么？

树状数组中每个元素 ft[i] 的含义？模拟求和，以求 [0, 6]（nums 是 1-indexed 的，也就是 [1, 7]）前缀和为例：

sum[1, 7] = ft[7] + ft[6] + ft[4]

  • 7 的二进制表示 111
  • 实际上，ft[7] = nums[7]
  • 实际上，ft[6] = nums[5] + nums[6]
  • 实际上，ft[4] = nums[1] + nums[2] + nums[3] + nums[4]

看了上面例子，对于 ft[i] 的本质基本有感觉了吧 🤔🤔

  • ft[i] 表示的是 nums[i - lowbit(i) + 1] + ... + nums[i] 的和！！！
  • ft[i] 维护的是区间 [i - lowbit(i) + 1, i] 的性质

lowbit(x) 在这期间的作用？

  • 求和期间：从上到下，快速跳转到下一个求和区间
  • 更新期间：从下到上，快速跳转到下一受影响的区间

树状数组（改造版）

实现原理

通过 “两次前缀和 + 减法” 的方式，树状数组可以查询任意区间所有元素和。 类似，任意区间所有元素的积也可以通过 “两次前缀积 + 除法” 的方式获得。

但对于bit_and、bit_or、bit_xor、max、min、gcd、lcm 等操作，由于没有第二步 “逆运算” 操作，导致无法对任意区间进行查询！！！

对于上面这些操作，树状数组真的就无能为力，只能到此为止了吗？别慌，接着往下看！

树状数组中，每个元素存储的是它附近一段区间的和，将查询区间拆分成多个这样连续区间的，再将这些区间的和加起来就得到了查询区间的和。

通过这种方式避免了 “逆运算” 操作，扩展了树状数组的应用场景

以查询区间 [3, 7] 最大值为例：

  • ft[7] 维护的范围是 [7, 7]，完全在查询范围内
  • ft[6] 维护的范围是 [5, 6]，完全在查询范围内
  • ft[4] 维护的范围是 [1, 4]，不完全在查询范围内，无法使用，先使用原数组元素代表 nums[4]
  • ft[3] 维护的范围是 [3, 3]，完全在查询范围内
  • max([3, 7]) = max(ft[7], ft[6], nums[4], ft[3])

封装及使用

支持区间 plus、multiple、bit_and、bit_or、bit_xor、max、min、gcd、lcm 等场景的单点更新，区间查询操作。

constexpr auto identity_value = nullptr;

template<typename T>
constexpr auto identity_value<std::plus, T> = T(0);

constexpr auto identity_value<std::bit_and, T> = ~T(0);

// std::bit_or partial variable template specialization

// std::maximum is function template, not type of functor

template<template<typename> class Op, typename T>
class FenwickTree {
public:
    explicit FenwickTree(const size_t n, const T& init_value = identity_value<Op, T>)
        : _ori(n, init_value), _ft(n + 1, identity_value<Op, T>) {
        linear_init();
    }

    explicit FenwickTree(const std::vector<T>& nums) : _ori(nums), _ft(nums.size() + 1, identity_value<Op, T>) {
        linear_init();
    }

    // 0-indexed noexcept
    const T& operator[](size_t idx) const noexcept { return _ori[idx]; }
    // 0-indexed with exception
    // const T& at(size_t idx) const { return ori.at(idx); }

    // 0-indexed
    bool update(size_t idx, const T& x) noexcept {
        _ori[idx] = _op(_ori[idx], x);
        idx++;
        while(idx < _ft.size()) {
            _ft[idx] = _op(_ft[idx], x);
            idx += lowbit(idx);
        }
        return true;
    }

    // 0-indexed [0, idx]
    T query(size_t idx) const {
        // change to 1-indexed
        idx++;
        // avoid Out-of-bounds access
        idx = std::min(idx, _ori.size());
        T res = identity_value<Op, T>;
        while(idx) {
            res = _op(res, _ft[idx]);
            idx -= lowbit(idx);
        }
        return res;
    }

    // 0-indexed [p, r]
    T query(size_t p, size_t r) const {
        // change to 1-indexed
        p++, r++;
        T res = identity_value<Op, T>;
        // special scene should return identity
        if(p > r || p > _ori.size()) return res;

        // void Out-of-bounds access
        r = std::min(r, _ori.size());
        while(p <= r) {
            if(p <= r - lowbit(r)) {
                res = _op(res, _ft[r]);
                r -= lowbit(r);
            } else {
                res = _op(res, _ori[--r]);
            }
        }
        return res;
    }

private:
    static size_t lowbit(const size_t x) { return x & -x; }
    void linear_init() {
        for(size_t i = 1; i < _ft.size(); ++i) {
            _ft[i] = _op(_ft[i], _ori[i - 1]);
            size_t idx = i + lowbit(i);
            if(idx < _ft.size()) _ft[idx] = _op(_ft[idx], _ft[i]);
        }
    }
    std::vector<T> _ori;
    std::vector<T> _ft;
    Op<T> _op;
};

在实现单点查询、区间查询时，需要注意：

  • 不要着急直接使用 clamp 处理边界，应该先判断特殊场景
    • 单点查询: idx == -2 时，实际应该返回单位元，如果先 clamp 到 idx == 0, 返回的并不是单位元
    • 区间查询: p > r || r < 0 || p >= ori.size() 时都应该返回单位元
  • 判断完特殊场景之后，需要进行 clamp，防止访问越界同时保证计算结果
    • 单点查询: idx == ori.size() + 3 时，计算前应该 clamp 到 idx == ori.size() - 1 防止访问越界
    • 区间查询: 特殊场景判断完之后，还有可能是 p < 0 && r >= ori.size() 等场景，所以还是要做 clamp 的

成员变量原数组 _ori 是必须的吗？

  • 对于 std::plus、std::multiple 这类存在 “逆运算” 的操作，可以从树状数组还原出原数组，也就不是必须的
  • 对于其它不存在 “逆运算” 的操作，无法还原出原数组，故是必须的

场景约束

树状数组只支持 “增量更新（update）”，不支持 “设置更新（set）” ！！！

考虑求区间最大值场景下，进行以下两次单点更新：

  • 单点更新下 nums[5] = 100，ft[5]、ft[6]、ft[8] 均受到影响，更新为 100，这没问题
  • 接着更新下 nums[5] = 99，按道理 ft[5]，ft[6]，ft[8] 均应更新为 99，但实际是不会更新的，这正是问题所在！

在最大/小值场景，要求更新方向只能朝着变大/小的方向修改！

  • 最大值查询：更新值必须大于等于旧值
  • 最小值查询：更新值必须小于等于旧值

拓展

离散化

给定一个数组 nums，0 <= nums[i] <= 10000，统计 [p, r] 区间内元素的个数？一种可行方案如下:

  • 创建一个长度为 10001 的一一映射数组 cnt，数字 x 出现的次数存储在 cnt[x] 中
  • 查询在区间 [p, r] 内的元素个数也就是 cnt[p] + cnt[p + 1] + ... + cnt[r]

如果元素的取值范围是 0 <= nums[i] <= 1000000000000 呢？开这么大的数组显然不现实！

在这个问题中，我们 只关注数值的相对大小，不关注具体数值，可以对数组进行 “排序 + 去重” 操作，然后用它们的下标代表原数字，这便是离散化：将无限空间中的有限个体映射到有限空间中的技术

  • 保持原数据相对大小关系
  • 将数据映射到更小的连续区间
  • 常用于处理值域大但数据量小的情况

示例：

  • 原始数据： [1000, 20000, 300, 1000, 50000]
  • 离散化后： [1, 2, 0, 1, 3]

使用时，先将 “原始的查询区间” 通过二分查找映射到 “离散后的查询区间”，然后在统计元素出现次数

// 排序
sort(nums.begin(), nums.end());

// 去重
nums.resize(unique(nums.begin(), nums.end()) - nums.begin());

// 查询区间离散化
int p1 = lower_bound(nums.begin(), nums.end(), p) - nums.begin();
int r1 = lower_bound(nums.begin(), nums.end(), r) - nums.begin();
