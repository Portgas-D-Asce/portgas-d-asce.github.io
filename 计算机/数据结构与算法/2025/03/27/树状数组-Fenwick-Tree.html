原数组 & 前缀和数组 & 差分数组 & 树状数组

设有原数组 $nums$:

  • 数组 $sum[i] nums[0] + nums[1] + … + nums[i]$ 称为 $nums$ 的前缀和数组
  • 数组 $dif[i] = nums[i] - nums[i - 1]$ 称为 $nums$ 的差分数组

前缀和数组、差分数组 只不过是原数组的不同表示形式。

  • 原数组可以在常数时间内访问任意元素！！！
  • 前缀和数组可以在常数时间获取任意区间的元素和！！！
  • 差分数组可以在常数时间内对任意区间内元素增加/减少固定数值！！！

前缀和、差分互为逆运算：

  • 原数组 是 差分数组 的前缀和数组！！！
  • 原数组 是 前缀和数组 的差分数组！！！

前缀和数组：

  • 直接存储了前缀和信息；
  • 构建时间复杂度： $O(n)$；
  • 查询原数组任意区间内元素的和： 时间复杂度 $O(1)$；
  • 修改原数组中某个元素的值需要对前缀和数组进行维护： 时间复杂度为 $O(n)$；

树状数组：

  • 间接存储了前缀和信息，被分为连续的多段
  • 构建时间复杂度： $O(n)$；
  • 查询原数组任意区间内元素的和： 实际上是查询两次前缀和，然后作差；时间复杂度 $O(logn)$；
  • 修改原数组中某个元素的值需要对树状数组进行维护： 时间复杂度为 $O(logn)$；

树状数组是一种 “兼顾单点修改与区间查询平衡的一种数据结构”

原理

先上代码，然后再根据代码反向理解：

  • $lowbit(x)$ 是什么，其作用又是什么？
  • 树状数组中每个元素 $ft[i]$ 的含义？
//nums 表示原数组，ft 表示 nums 的树状数组
//同前缀和数组，树状数组也需要补充一个前置 0
class FenwickTree{
private:
    vector<int> ft;
    int lowbit(int x) {
        return x & (-x);
    }
public:
    FenwickTree(int n) {
        ft.resize(n + 1, 0);
    }
    //第 idx 个元素 + x
    //nums[idx] + x when 1-indexed
    //nums[idx - 1] + x when 0-indexed
    void add(int idx, int x) {
        while(idx < ft.size()) {
            ft[idx] += x;
            idx += lowbit(idx);
        }
    }
    //前缀和查询
    int sum(int idx) {
        int res = 0;
        while(idx) {
            res += ft[idx];
            idx -= lowbit(idx);
        }
        return res;
    }
};

lowbit 函数

$lowbit$ 是什么？

  • $lowbit(x)$：$x$ 的二进制表示最后一位 $1$ 的位权
  • $lowbit(x) = x\ \&\ (-x)$
x 的二进制表示为：0...1000000
x 的反码为：     1...0111111
x 的补码为：     1...1000000（也就是 -x 的二进制表示）
x & (-x)：      0...1000000

理解

原理 + 实现的时候基于 1-indexed 更好理解，接口封装时 0-indexed 使用起来更方便！！！

树状数组中每个元素 $ft[i]$ 的含义？模拟求和（为了方便描述，这里假设原数组 $nums$ 下标也从 $1$ 开始）：

  • $sum[7] = ft[7] + ft[6] + ft[4]$
    • 7 的二进制表示 $111$
    • 实际上，$ft[7] = nums[7]$
    • 实际上，$ft[6] = nums[5] + nums[6]$
    • 实际上，$ft[4] = nums[1] + nums[2] + nums[3] + nums[4]$
  • 看了上面例子，对于 $ft[i]$ 的本质基本有感觉了吧 🤔🤔
    • $ft[i]$ 表示的是 $nums[i - lowbit(i) + 1] + … + nums[i]$ 的和！！！
    • $ft[i]$ 维护的是区间 $[i - lowbit(i) + 1, i]$ 的性质

$lowbit(x)$ 在这期间的作用？

  • 求和期间：从上到下，快速跳转到下一个求和区间
  • 更新期间：从下到上，快速跳转到下一受影响的区间

区间最值

What？？？树状数组可以求任意区间和是可以理解的：求两次前缀和，做差即可得到区间和。树状数组求任意区间最值？！！

一开始也是不相信的，然而事实是，在特定场景下是可行的，而且效果还不错。

特定场景：求区间最小/大值时，要求修改方向只能朝着变小/大的方向修改（线段树是没有这要求的），考虑区间最大值：

  • 单点更新下 $nums[5] = 100$，$ft[5]$，$ft[6]$，$ft[8]$ 均受到影响，更新为 $100$，这没问题
  • 接着更新下 $nums[5] = 99$，按道理 $ft[5]$，$ft[6]$，$ft[8]$ 均应更新为 $99$，但实际是不会更新的，这正是问题所在！

实现

这里以求区间最大值为例，核心思想：

  • 更新的时候不仅要维护树状数组 $ft$，还要维护原数组 $nums$
  • 查询的时候 $[p, r]$ 最值朴素做法：从起点到终点逐个遍历，取最大值即可，可以使用 $ft$ 对其进行优化
    • 若 $ft[i]$ 所管理的区间在查询区间范围内：用 $ft[i]$ 代表其所管理的区间即可
    • 如果 $ft[i]$ 所管理的区间并没有完全在查询区间内，回到朴素做法即可，用原是数组中的元素做比较即可

示例：查询区间 $[3, 7]$

  • $ft[7]$ 管理的范围是 $[7, 7]$，完全在查询范围内
  • $ft[6]$ 管理的范围是 $[5, 6]$，完全在查询范围内
  • $ft[4]$ 管理的范围是 $[1, 4]$，不完全在查询范围内，使用原数组元素代表 $nums[4]$
  • $ft[3]$ 管理的范围是 $[3, 3]$，完全在查询范围内
  • $max([3, 7]) = max(ft[7], ft[6], nums[4], ft[3])$

离散化

维护离散区间

常规：下标区间是连续的，维护区间 [5, 6] 之间的最值/和

如果下标区间不连续呢：

  • 范围小的话，声明一个数组，当成连续的就可以了
  • 范围特别大，就没办法声明数组了

解决办法：

  • 对出现过的下标进行排序，排完序后，去重
  • 每次先用二分查找，查找到对应的连续下标
//排序
sort(diff.begin(), diff.end());
//去重
diff.resize(unique(diff.begin(), diff.end()) - diff.begin());
Fenwick fw(diff.size());

long long mx = LLONG_MIN;
for(int i = 0; i < n; ++i) {
    //离散下标转化为连续下标
    int j = lower_bound(diff.begin(), diff.end(), nums[i] - i) - diff.begin();
    long long temp = fw.query(j + 1) + nums[i];
    fw.update(j + 1, temp);
    mx = max(mx, temp);
}

封装

支持区间 和、与和、或和、异或和、最大/小值（gcd？？？？）等场景的 单点更新，区间查询操作。

单点更新的输入不是 “修改后的新值”，而是是 “变化值“ 是在原来数字的基础上进行更新，有些操作中（除了和、异或和场景）是移除不掉旧有元素影响的，使用起来具有约束，以最值场景为例:

  • 最大值查询：更新值必须大于等于旧值
  • 最小值查询：更新值必须小于等于旧值
#include <vector>
#include <algorithm>
#include <iostream>
#include <limits>


// if you like this code, don't delete this, thanks  --from pk
// 单位元变量模板声明(C++14)
template<template<typename> class Op, typename T>
constexpr auto identity_value = nullptr;

// 加法特化
template<typename T>
constexpr auto identity_value<std::plus, T> = T(0);

// 乘法特化
template<typename T>
constexpr auto identity_value<std::multiplies, T> = T(1);

// 按位与特化
template<typename T>
constexpr auto identity_value<std::bit_and, T> = ~T(0);

// 按位或特化
template<typename T>
constexpr auto identity_value<std::bit_or, T> = T(0);

// 按位异或特化
template<typename T>
constexpr auto identity_value<std::bit_xor, T> = T(0);

// 最大值函数对象
template<typename T> struct maximum  {
    constexpr T operator()(const T& lhs, const T& rhs) const { return std::max(lhs, rhs); }
};
// 最大值特化, 浮点型用 lowest()
template<typename T> constexpr auto identity_value<maximum , T> =
    std::is_floating_point<T>::value ? std::numeric_limits<T>::lowest() : std::numeric_limits<T>::min();

// 最小值函数对象
template<typename T> struct minimum {
    constexpr T operator()(const T& lhs, const T& rhs) const { return std::min(lhs, rhs);}
};
// 最小值特化
template<typename T> constexpr auto identity_value<minimum, T> = std::numeric_limits<T>::max();

template<template<typename> class Op, typename T>
class FenwickTree {
public:
    explicit FenwickTree(const int n, const T& init_value = identity_value<Op, T>) : ori(n, init_value), ft(n + 1, identity_value<Op, T>) {
        linear_init();
    }
    
    explicit FenwickTree(const std::vector<T>& nums) : ori(nums), ft(nums.size() + 1, identity_value<Op, T>) {
        linear_init();
    }

    bool update(int idx, T x) {
        int n = ori.size();
        if(idx < 0 || idx >= n) return false;
        
        ori[idx] = op(ori[idx], x);
        idx++;
        while(idx < ft.size()) {
            ft[idx] = op(ft[idx], x);
            idx += lowbit(idx);
        }
        return true;
    }

    // 0-indexed [0, idx]
    T query(int idx) const {
        // change to 1-indexed
        idx++;
        T res = identity_value<Op, T>;
        // special scene should return identity
        if(idx < 1) return res;
        // avoid Out-of-bounds access
        if(idx > ori.size()) idx = ori.size();
        while(idx) {
            res = op(res, ft[idx]);
            idx -= lowbit(idx);
        }
        return res;
    }

    // 0-indexed [p, r]
    T query(int p, int r) const {
        // change to 1-indexed
        p++, r++;
        int n = static_cast<int>(ori.size());
        T res = identity_value<Op, T>;
        // special scene should return identity
        if(p > r || r < 1 || p > n) return res;
        
        // void Out-of-bounds access
        if(p < 1) p = 1;
        if(r > n) r = n;
        while(p <= r) {
            if(p <= r - lowbit(r)) {
                res = op(res, ft[r]);
                r -= lowbit(r);
            } else {
                res = op(res, ori[--r]);
            }
        }
        return res;
    }

private:
    static int lowbit(const int x) { return x & (-x); }
    void linear_init() {
        int n = ori.size();
        for(int i = 1; i <= n; ++i) {
            ft[i] = op(ft[i], ori[i - 1]);
            int idx = i + lowbit(i);
            if(idx <= n) ft[idx] = op(ft[idx], ft[i]);
        }
    }
    std::vector<T> ori;
    std::vector<T> ft;
    Op<T> op;
};

int main() {
    std::cout << identity_value<maximum, int> << std::endl;
    std::cout << identity_value<minimum, int> << std::endl;
    std::cout << identity_value<maximum, unsigned int> << std::endl;
    std::cout << identity_value<minimum, unsigned int> << std::endl;
    std::cout << identity_value<maximum, long long> << std::endl;
    std::cout << identity_value<minimum, long long> << std::endl;
    std::cout << identity_value<maximum, unsigned long long> << std::endl;
    std::cout << identity_value<minimum, unsigned long long> << std::endl;
    std::cout << identity_value<maximum, float> << std::endl;
    std::cout << identity_value<minimum, float> << std::endl;
    std::cout << identity_value<maximum, double> << std::endl;
    std::cout << identity_value<minimum, double> << std::endl;

    FenwickTree<std::plus, int> ft(10);

    FenwickTree<minimum, int> ft1(10);

    ft.update(0, 1);
    ft.update(1, 2);
    ft.update(9, 3);
    std::cout << ft.query(9) << std::endl;


    return 0;
}

在实现单点查询、区间查询时，需要注意：

  • 不要着急直接使用 clamp 处理边界，应该先判断特殊场景
    • 单点查询: idx == -2 时，实际应该返回单位元，如果先 clamp 到 idx == 0, 返回的并不是单位元
    • 区间查询: p > r || r < 0 || p >= ori.size() 时都应该返回单位元
  • 判断完特殊场景之后，需要进行 clamp，防止访问越界同时保证计算结果
    • 单点查询: idx == ori.size() + 3 时，计算前应该 clamp 到 idx == ori.size() - 1 防止访问越界
    • 区间查询: 特殊场景判断完之后，还有可能是 p < 0 && r >= ori.size() 等场景，所以还是要做 clamp 的
