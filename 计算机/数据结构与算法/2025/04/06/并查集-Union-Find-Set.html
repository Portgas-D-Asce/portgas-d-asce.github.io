简介

并查集：用于处理不相交集合问题的一种数据结构。定义在其上的两个操作：

  • Find： 查找节点所属于的集合
  • Unite： 合并两个不相交的集合

典型应用当属 Kruskal 算法，优化方式：

  • 路径压缩： 让查找路径上的所有节点均指向根节点，提高下一次查找效率；
  • 启发式合并： 基本没啥用，把 “秩” 小的集合合并到 “秩” 大的集合，使得合并后集合的 “秩” 处于较小状态，有利于查找。 （）

注意：update 操作显然会改变秩的大小，但代码并没有更新，“秩”（rank）可以理解为树高度的上界，并不是精确高度，仅用于指导合并！！！

class UnionFind {
public:
    explicit UnionFind(const int n, const int domain = 1) : _count(n * domain), _parent(n * domain),
        _rank(n * domain), _size(n * domain, 1) {
        std::iota(_parent.begin(), _parent.end(), 0);
    }

    int find(const int x) const {
        // return _parent[x] == x ? x : _parent[x] = find(_parent[x]);

        // firstly, find root
        int root = x;
        while (root != _parent[root]) root = _parent[root];

        int cur = x;
        // secondly, path reduce
        while (cur != root) {
            const int tmp = _parent[cur];
            _parent[cur] = root;
            cur = tmp;
        }
        return root;
    }

    bool unite(const int u, const int v) {
        const int pu = find(u), pv = find(v);
        if (pu == pv) return false;

        if (_rank[pu] > _rank[pv]) {
            _parent[pv] = pu;
            _size[pu] += _size[pv];
        } else if (_rank[pu] < _rank[pv]) {
            _parent[pu] = pv;
            _size[pv] += _size[pu];
        } else {
            _parent[pv] = pu;
            _size[pu] += _size[pv];
            _rank[pu]++;
        }
        _count--;
        return true;
    }

    // u and v belong same set?
    bool connected(const int u, const int v) const { return find(u) == find(v); }

    // count of components
    int count() const { return _count; }

    // the set's size u belong to
    int size(const int u) const { return _size[find(u)]; }
private:
    int _count; // cnt of set
    mutable std::vector<int> _parent; // array of parent node
    std::vector<int> _rank; // rank, tree's height
    std::vector<int> _size; // set's size
};

扩展域并查集

经典题目：

  • 关押罪犯

关押罪犯：

  • 合并 (a, b + 1n), (a + 1n, b) 表示建立敌对关系

食物链

  • 合并 (a, b + 1n), (a + 1n, b) 表示建立捕食关系
  • 合并 (a, b + 2n), (a + 2n, b) 表示建立天敌关系

1. 核心思想：从“关系”到“集合”

普通并查集擅长维护 “朋友的朋友是朋友” 这种无冲突的连通性关系。

但当问题中出现了 对立关系（比如“敌人”、“不同性别”、“不同颜色”）时，仅仅一个集合就不够用了。扩展域并查集的核心思想是：

将每个实体拆分成多个逻辑上的点，用这些点之间的关系来代表实体的复杂关系。

最经典的情况是处理 只有两种对立关系 的问题（例如：是/否、真/假、A队/B队）。这时，我们为每个实体 i 创建 两个 逻辑点：

  • i_A：表示 “i 属于 A 类”
  • i_B：表示 “i 属于 B 类”

这两个点是互斥的，不能同时成立。

2. 它如何工作？

并查集的作用是 将必须同时发生的事件（逻辑点）合并到同一个集合中。

我们通过一个具体的例子来说明。假设有一个江湖，我们要处理侠客之间的朋友和敌人关系。

  • 规则1：朋友的朋友是朋友。
  • 规则2：敌人的敌人是朋友。

我们为每个侠客 x 创建两个域（逻辑点）：

  • x_friend：表示 x 是朋友阵营的
  • x_enemy：表示 x 是敌人阵营的

(注意：在实际编程中，我们用数字来表示这些点。例如，对于侠客 i，i 本身代表 i_friend，i + n 代表 i_enemy，其中 n 是总人数。)

场景一：建立朋友关系

操作： x 和 y 是朋友。

逻辑推导：

  1. 如果 x 是朋友阵营的，那么 y 也必须是朋友阵营的。 => 合并 x_friend 和 y_friend
  2. 如果 x 是敌人阵营的，那么 y 也必须是敌人阵营的。 => 合并 x_enemy 和 y_enemy

代码：

union(x, y)          # 合并 friend 域
union(x + n, y + n)  # 合并 enemy 域

场景二：建立敌人关系

操作： x 和 y 是敌人。

逻辑推导：

  1. 如果 x 是朋友阵营的，那么 y 必须是敌人阵营的。 => 合并 x_friend 和 y_enemy
  2. 如果 x 是敌人阵营的，那么 y 必须是朋友阵营的。（敌人的敌人是朋友） => 合并 x_enemy 和 y_friend

代码：

union(x, y + n)  # x是朋友 => y是敌人
union(x + n, y)  # x是敌人 => y是朋友

3. 如何检测矛盾？

矛盾发生在 两个本应互斥的逻辑点被合并到了同一个集合。

继续上面的例子，当我们处理一条新关系时：

  • 如果声明 x 和 y 是朋友，我们需要检查： x_friend 和 y_enemy 是否在同一个集合？或者 x_enemy 和 y_friend 是否在同一个集合？ 如果是，就说明产生了矛盾。因为“是朋友”和“是敌人”不能同时成立。

  • 如果声明 x 和 y 是敌人，我们需要检查： x_friend 和 y_friend 是否在同一个集合？或者 x_enemy 和 y_enemy 是否在同一个集合？ 如果是，就说明产生了矛盾。

代码检查（以敌人关系为例）：

if find(x) == find(y) or find(x + n) == find(y + n):
    print("矛盾！这条敌人关系不成立。")

4. 经典例题：食物链

这是理解扩展域的最佳题目。有三种动物 A, B, C，构成一个循环食物链：A吃B，B吃C，C吃A。

我们用 三个域 来表示每个动物 i 的三种可能身份：

  • i_self (域 1)：表示 i 是 A 类
  • i_food (域 2)：表示 i 是 B 类 (A的食物)
  • i_enemy (域 3)：表示 i 是 C 类 (B的食物，同时吃A)

规则：

  1. 如果 x 和 y 是同类：
    • 那么 x_self 和 y_self 是同类。
    • x_food 和 y_food 是同类。
    • x_enemy 和 y_enemy 是同类。
    • union(x_self, y_self), union(x_food, y_food), union(x_enemy, y_enemy)
  2. 如果 x 吃 y：
    • 那么 x_self 和 y_food 是同类 (A吃B)。
    • x_food 和 y_enemy 是同类 (B吃C)。
    • x_enemy 和 y_self 是同类 (C吃A)。
    • union(x_self, y_food), union(x_food, y_enemy), union(x_enemy, y_self)

矛盾检测：

  • 当说 x 和 y 是同类时，检查 x_self 和 y_food 或 x_self 和 y_enemy 是否在同一集合。如果是，则矛盾（因为x是A，y就不能是B或C）。
  • 当说 x 吃 y 时，检查 x_self 和 y_self 或 x_self 和 y_enemy 是否在同一集合。如果是，则矛盾（因为A不能吃A，也不能吃C）。

总结

特性 普通并查集 扩展域并查集
核心 维护连通性 维护逻辑关系（相容、互斥）
元素 每个实体一个点 每个实体拆成 k 个逻辑点 (k是关系种类数)
操作 union(a, b) 同时进行多个 union，将相容的逻辑点连接
矛盾检测 通常不直接处理 检查互斥的逻辑点是否被意外连接
适用场景 图连通性、无向图环检测 种类并查集、二分图判断、逻辑推理问题

一句话理解扩展域并查集： 它通过为每个实体创建多个“平行世界”（域），并在这些世界之间建立“如果…那么…”的链接，从而推理出复杂的关系和矛盾。
