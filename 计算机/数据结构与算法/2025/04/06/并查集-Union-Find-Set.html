概述

并查集：用于处理不相交集合问题的一种数据结构。定义在其上的两个操作：

  • find: 查找节点所属于的集合
  • unite: 合并两个不相交的集合

典型应用当属 Kruskal 算法。

优化方式：

  • 路径压缩： 让查找路径上的所有节点均指向根节点，提高下一次查找效率；
  • 启发式合并：
    • 基本没啥用，把 “秩” 小的集合合并到 “秩” 大的集合，使得合并后集合的 “秩” 处于较小状态，有利于查找。
    • update 操作显然会改变秩的大小，但代码并没有更新，“秩”（rank）可以理解为树高度的上界，并不是精确高度，仅用于指导合并！！！

普通并查集

代码封装

class UnionFind {
public:
    explicit UnionFind(const int n) : _components(n), _parent(n), _rank(n), _size(n, 1) {
        std::iota(_parent.begin(), _parent.end(), 0);
    }

    int find(const int x) const {
        // return _parent[x] == x ? x : _parent[x] = find(_parent[x]);

        // firstly, find root
        int root = x;
        while (root != _parent[root]) root = _parent[root];

        int cur = x;
        // secondly, path reduce
        while (cur != root) {
            const int tmp = _parent[cur];
            _parent[cur] = root;
            cur = tmp;
        }
        return root;
    }

    bool unite(const int u, const int v) {
        const int pu = find(u), pv = find(v);
        if (pu == pv) return false;

        if (_rank[pu] > _rank[pv]) {
            _parent[pv] = pu;
            _size[pu] += _size[pv];
        } else if (_rank[pu] < _rank[pv]) {
            _parent[pu] = pv;
            _size[pv] += _size[pu];
        } else {
            _parent[pv] = pu;
            _size[pu] += _size[pv];
            _rank[pu]++;
        }
        _components--;
        return true;
    }

    // u and v belong same set?
    bool connected(const int u, const int v) const { return find(u) == find(v); }

    // count of components
    int components() const { return _components; }

    // the component's size u belong to
    int size(const int u) const { return _size[find(u)]; }
private:
    int _components; // cnt of components
    mutable std::vector<int> _parent; // array of parent node
    std::vector<int> _rank; // rank, tree's height
    std::vector<int> _size; // set's size
};

问题背景

扩展域并查集（也称为种类并查集或关系并查集）是普通并查集的强力扩展，用于处理元素之间具有多种可能关系的问题。它不仅告诉你两个元素是否“在同一个集合”，还能表达它们之间的具体关系（如朋友、敌人、捕食者、被捕食者等），并自动维护关系的传递性

核心原理

核心思想：扩展域并查集的核心是将元素拆分为多个状态节点（域划分），通过合并节点来表达关系（建立关系）

合并规则与冲突检测：

  • 扩展域并查集的关键是根据给定的关系，确定哪些域应该被合并。合并规则必须保证关系的对称性和传递性。
  • 同时，在合并前要检查是否与已有信息矛盾，也就是同一元素的不同状态被连接在一起了。

扩展域并查集不要求关系一定要闭环：每条关系只是添加一些节点之间的连通性约束，并不会强制要求所有可能的关系必须出现

域的数量与编号

域的数量取决于关系种类的个数。若关系构成一个k元环（即关系可循环传递），则通常需要k个域。一般地，对于有k种可能状态的问题，每个元素拆成k个节点。

编号方式：常用方法是将第i个域的第j个元素（0 ≤ i < k）映射为全局唯一整数：

  • 初始化：并查集大小为 n * k，每个节点的父节点指向自己。
  • [0, n - 1]个位置表示第一个域的所有节点
  • [n, 2n - 1] 个位置表示第二个域的所有节点
  • …
id = i * k + j

典型示例

朋友/敌人关系：

  • 划分出两个域：朋友域 friend 和 敌人域 enemy，对于元素 a，拆分出两个节点 a_friend 和 a_enemy，两个节点隐含冲突关系
  • 建立等价关系：a 和 b 是朋友，则 unite(a_friend, b_friend), unite(a_enemy, b_enemy)
  • 建立敌人关系：b 和 c 是敌人，
    • 对称性：b 是 c 的敌人，c 也是 b 的敌人，unite(b_friend, c_enemy), unite(c_friend, b_enemy)
    • 传递性：a 和 c 也是敌人

捕食链关系：A 吃 B，B 吃 C，C 吃 A

  • 划分出三个域：自身域 self、天敌域 predator、猎物域 prey
  • 建立等价关系：a 和 b 是同一物种，则 unite(a_self, b_self), unite(a_predator, b_predator), unite(a_prey, b_prey)
  • 建立捕食关系：b 吃 c
    • 对称性：b 吃 c, c 被 b 吃，unite(b_self, c_predator), unite(c_self, b_prey), unite(c_prey, b_predator)
    • 传递性：a 也吃 c

闭环条件场景，连通分量的个数一定是域的倍数。

数学本质：群作用与等价关系

将每个元素的k个域看作集合 S = {0,1,…,k-1}。关系 x R y 可以定义为一个置换 f: S → S，表示若x处于状态p，则y必须处于状态f(p)。那么扩展域并查集本质上是在维护一个无向图，节点是 (元素, 状态)。合并操作相当于添加边 (x,p) — (y, f(p))（对所有p）。由于并查集是无向的，添加一条边意味着两个节点连通，因此所有受约束的状态都被绑定在一起。

当加入多条关系后，每个连通分量代表一组“兼容”的状态赋值。如果某个元素的两个不同状态出现在同一连通分量中（例如 (x, p) 和 (x, q) 连通），则说明存在矛盾，因为同一个元素不能同时处于两种状态。

因此，冲突检测就是检查是否存在某个元素的不同状态被连在了一起。

动态开点与稀疏场景

当k很大（如k=10^5）但n较小，或者关系数量很少时，直接开 n*k 数组可能内存爆炸。此时可以用哈希表或动态分配来存储并查集节点。每次需要用到某个域时才创建节点，并记录其父节点。但注意，动态开点会带来常数增加，且需要自己管理节点映射。

一种常用技巧是将 (x, t) 对映射为一个唯一整数（如用 pair<int,int> 作为key），然后用 unordered_map 存储其父节点。合并时只涉及实际出现的节点。

与带权并查集的关系

带权并查集为每个节点维护一个到根节点的权值（通常是一个模数），表示该节点与根的关系。例如在食物链问题中，权值0表示同类，1表示吃根，2表示被根吃。通过路径压缩时更新权值，同样能解决问题。

对比：

扩展域：思路直观，无需复杂的权值运算，实现简单，但空间占用较大（O(kn)）。 带权并查集：空间节省（O(n)），但需要推导权值运算法则（模加法/减法），容易出错，尤其在关系复杂时。 本质联系：实际上，扩展域并查集可以看作带权并查集的一种“显式”展开。每个元素的不同域对应带权并查集中该元素可能具有的权值。当合并时，扩展域合并的是具体状态节点，而带权并查集则通过调整权值来保持一致性。二者在数学上是等价的，可以互相转换。

扩展域在2-SAT问题中的类比

2-SAT问题中，每个变量有两个状态（真/假），约束是子句形式。扩展域并查集可以处理一些特殊的2-SAT变体，特别是当约束为“x与y同值”或“x与y异值”时，可以转化为二域模型。实际上，二域并查集就是2-SAT的一种特例解法。

但2-SAT更一般，需要强连通分量；而扩展域并查集只能处理具有传递性的关系（即关系构成等价类），不能处理任意布尔表达式。

复杂关系的合并规则推导

当问题不是简单的循环关系时，如何推导合并规则？例如，有四种状态，关系“x和y是同类”合并相同域，“x和y是敌对”则合并 x_i 与 y_j 其中 j = i xor 1？这种可能需要根据问题的定义仔细分析。

一般方法：将每种关系看作一个二元约束，写出所有可能的对应情况，然后合并所有对应的节点对。同时要保证合并后不出现某个元素的两个不同状态连通。这实际上是在构建一个约束图，然后检查是否可满足。
